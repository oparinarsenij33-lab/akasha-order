# Акаша 2.5

PHP-сайт учебного кабинета Ордена: уроки, домашние задания, ответы, личные сообщения, успеваемость, ручные корректировки, блокировки и управление пользователями.

## Что исправлено в 2.5

Главная причина «частично пропавших» стилей была не только в CSS. Браузер получал новый `script.js`, но продолжал использовать старый `style.css` из Service Worker и кэша. Поэтому новые элементы — поиск, нижняя навигация и карточки успеваемости — отображались как обычные неоформленные HTML-кнопки и текст.

В этой версии:

- все основные файлы получили новые имена, поэтому старый кэш не может их подменить;
- старый Service Worker автоматически удаляется;
- для HTML, CSS и JavaScript добавлены заголовки без кэширования;
- добавлен отдельный стабильный слой `ui-2.5.css`, который перекрывает конфликтующие старые правила;
- полностью проверены мобильные экраны шириной 390 px;
- нет горизонтальной прокрутки в участниках, Совете Мастеров, успеваемости и админ-панели;
- длинные имена отображаются полностью и переносятся по словам;
- нижнее меню всегда собирается в одну строку из пяти кнопок;
- поиск и счётчик результатов оформлены одинаково во всех списках;
- группы без совпадений скрываются при поиске;
- успеваемость на телефоне показывается карточками, на компьютере — таблицей;
- админ-панель и формы добавления пользователя адаптированы под телефон;
- модальные окна используют обычные `input` и `textarea`, поэтому работает системная клавиатура, каретка, выделение, копирование и вставка;
- плавающая кнопка настроек скрыта на телефоне: настройки доступны через пункт **«Ещё»**;
- обновлены версии клиента и PHP API до 2.5.0.

## Обновление существующего сайта

1. Сделайте резервную копию папки `data`.
2. Загрузите всё содержимое архива в папку сайта с заменой файлов.
3. **Не удаляйте** существующий файл базы:
   - `data/akasha.sqlite`, или
   - `data/akasha-store.json`.
4. Старые файлы больше не используются и их можно удалить:
   - `style.css`;
   - `script.js`;
   - `storage-client.js`;
   - `config.js`.
5. Откройте сайт обычным обновлением страницы. Версия 2.5 сама удалит старый Service Worker и кэш Акаши.

Основные файлы интерфейса теперь называются:

- `base-2.5.css`;
- `ui-2.5.css`;
- `app-2.5.js`;
- `storage-2.5.js`;
- `config-2.5.js`.

## Установка на новый хостинг

Требования:

- PHP 8.1 или новее;
- PHP должен иметь право записи в папку `data`;
- рекомендуется расширение `pdo_sqlite`, но при его отсутствии автоматически используется JSON-хранилище;
- для публичного сайта рекомендуется HTTPS.

Порядок установки:

1. Загрузите содержимое архива в `public_html`, `www` или другую корневую папку сайта.
2. Проверьте, что папка `data` доступна PHP для записи. Обычно подходят права `755` или `775`.
3. Откройте адрес сайта.
4. При первом запросе база и стартовые аккаунты создадутся автоматически.

Для Apache защита папки `data` уже находится в `data/.htaccess`. Для Nginx прямой доступ к этой папке нужно запретить в конфигурации сервера.

## Тестовый вход

Верховный Магистр:

- имя: `Аранэль Хальдарон`;
- пароль: `A1H23`.

Перед публичным запуском обязательно смените стартовые пароли.

## Добавление пользователя

1. Войдите под Магистром или Верховным Магистром.
2. Откройте **«Админ-панель»**.
3. Нажмите **«Добавить пользователя»**.
4. Заполните имя, ранг, наставника и временный пароль.
5. Сохраните форму.

В карточке пользователя доступны изменение профиля, смена пароля, блокировка и разблокировка.

## Резервная копия

Для резервной копии достаточно сохранить один из файлов:

- `data/akasha.sqlite`;
- `data/akasha-store.json`.

В нём находятся пользователи, задания, ответы, сообщения, отметки уроков, корректировки и журнал действий.
